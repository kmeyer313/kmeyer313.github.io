Selectors

SELECTOR MEANING CSS EXAMPLE HTML SYNTAX
UNIVERSAL Applies to all elements in the document * {} n/a
TYPE Matches element names h1, h2, h3 {} <h1>, <h2>, <h3>
CLASS Matches an element whose class attribute has a value that matches the one specified after the period (or full stop) symbol .value {} OR p.value {} <section class="value">
ID Matches an element whose id attribute has a value that matches the one specified after the pound or hash symbol #value {} <section id="value">
CHILD Matches an element that is a direct child of another li>a {} <h1>, <h2>, <h3>
DESCENDANT Matches an element that is a descendent of another specified element (not just a direct child of that element) p a {} <p><a></a></p>
ADJACENT SIBLING Matches an element that is the next sibling of another h1+p
GENERAL SIBLING Matches an element that is a sibling of another, although it does not have to be the directly preceding element h1~p

Boxes

BLOCK-LEVEL ELEMENTS start on a new line (<h1>, <p>, <ul>, <li>)

INLINE ELEMENTS flow in between surrounding text (<img>, <b>, <i>)

You can control how much space each box takes up by setting the width of the boxes (sometimes the height, too).

To separate boxes, you can use borders, margins, padding, and background colors.

USING DISPLAY

display: inline;

causes a block-level element to act like an inline element

display: block;

causes an inline element to act like a block-level element

display: inline-block;

causes a block-level element to flow like an inline element, while retaining other features of a block-level element

display: none;

hides an element from the page; the element acts as though it is not on the page at all

have joining strokes or other cursive characteristics or other cursive characteristics, such as handwriting styles (ie. Comic Sans MS, Monotype Corsiva)

usually decorative and often used for titles; not designed for long bodies of text (ie. Impact, Haettenschweiler)

POSITIONING SCHEMAS

  • Normal Flow This is the default behavior where every block-level element appears on a new line.
  • Relative This moves an element from the position it would be in normal flow, shifting it to the top, right, bottom, or left of where it would have been placed. This does not affect the position of the surrounding elements.
  • Absolute This positions the element in relation to its containing element. It is taken out of normal flow, not affecting the position of any surrounding elements. These elements move as users scroll up and down the page.
  • Fixed A form of absolute positioning but it positions the element in relation to the browser window. Elements with fixed positioning do not affect the position of surrounding elements and do not move when the user scrolls up or down the page.
  • Floating This allows you to take that element out of normal flow and position it to the far left or right of a containing box. The floated element becomes a block-level element around which other content can flow.
  • z-index When you move any element from normal flow, boxes can overlap. This property allows you to control which box appears on top.

FIXED WIDTH VS LIQUID

  • Fixed width layout designs do not change size as the user increases or decreases the size of their browser window. Measurements tend to be given in pixels.
  • Liquid layouts designs stretch and contract as the user increases or decreases the size of their browser window. They tend to use percentages Measurements tend to be given in pixels.
