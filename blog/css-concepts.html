November 30, 2015

What are the differences between relative, absolute, fixed, and static positioning?

First and foremost: Make sure you look over your work to make sure you didn't forget important little pieces that can impact your entire page! You know when you are writing a paragraph in a word processor and you forget to include punctuation here and there like a period at the end of a sentence? It can throw off the format and overall flow of the paper, right? In the same way, if you forget end syntax when writing CSS like a semicolon or a closing curly bracket, the way elements are positioned can be completely thrown off. So before you go any further with trying to position your page in CSS, validate your code here to make sure you haven't forgotten any important syntax. Many frustrations disappeared when I found little errors like forgetting to include that end syntax!

Now. It's going to get a little complicated and I don't consider myself an expert yet, but here are a few things I have found to be helpful when working with positioning. Here are some notes I've surmised about each applied to one of our positioning challenges (see screenshot below when referenced):

STATIC (Div 1)

This is the default position, that means this is what the elements will automatically take on for their position on a page. (Being the default, you do not have to delagate the position in your CSS stylesheet.)

A static element will automatically assume the width of the page or the browser window. See the "Div 1" box in the image below. It takes up the whole width of the page.

In static, block elements naturally stack on top of each other. This is the natural flow of the page. If Div 2, Div 3, and Div 4 were all static, they would line up under Div 1, looking just like it but with their different colors.

You cannot tell a static element how far to go from the top, right, bottom, or left of the page. To do this, we need to move on to relative positioning.

However, it is important to know, that blocks in static can have margins. In the example below, the Div 1 element DOES have a bottom margin of 20 pixels that will affect Div 2, as you will see in the next section.

RELATIVE (Div 2)

Relative positioned elements want to follow the natural flow of the page like they do in static. However, we can be more specific about where we want a relative positioned element to be by giving them directional properties (how far from left, top, right, bottom).

In the image below, it is hard to see at a glance that Div 2 is in the natural flow of the page, but knowing what is going on behind the scenes will help.

Div 2 has some other properties that have been delegated to it that make it look so far away from Div 1:

  1. It is given dimensions of width and height, 200 pixels for each.
  2. It has been pushed to the left 50 pixels.
  3. Think carefully now, Div 2 is set to be 0 pixels from the top of the preceeding element in the flow -Div 1- but why is it so far away? Do you remember at the end of the Static section when I said that Div 1 has a bottom margin of 20 pixels? Well, that's why Div 2 is so far away from Div 1! It is respecting the properties of the previous element in the flow, therefore the top of Div 2 is starting exactly at the bottom of the margin of Div 1.

ABSOLUTE (Div 3)

An element with absolute positioning ABSOLUTELY wants to stay where it has been assigned in relation to the browser window.

It does not impact the flow of static or relative positioned elements and will sit on top of other elements, as you can see below, Div 3 is sitting on top of Div 2.

It can be given dimensions of width and height, and also directional properties of top, right, bottom, left.

Things start to get tricky when an element with absolute position is nested in another element that has relative, absolute, or fixed value. But basically, it will absolutely position itself relative to its parent element.

FIXED (Div 4)

An element with a fixed position is much like an element in absolute position but it will never respect any of its parents positions.

It will always position itself in relation to the browser window.

In the image below, this isn't captured, but if you were to scroll up and down the page where the div boxes are, Div 4 would stay in that exact same location.


Well this is what I have figured out so far. I have found that floats, padding, and margins really throw me off when it comes to working with positioning. I'm looking forward to learning more about those and become more skilled with positioning!
